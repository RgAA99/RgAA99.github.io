一、什么是hexo

hexo是基于nodejs的静态网页生成器，可以用来写博客，展示产品等。

(1) 安装过程

  1. 安装node和git
    • git是为了从hexo库中拉取以及将本地部署推到github.io
  2. 按命令执行
    • 安装hexo和相关命令
      npm install -g hexo-cli
    • 在指定下初始化hexo
      hexo init <folder>
    • 报错后进文件夹重新install
      cd <folder>
      npm install
    • 生成网站页面(生成在public目录下)
      hexo g
    • 部署至本地server
      hexo s
  3. 写作
    • 创建一篇新文章(默认模板是post,文章的模板参照scaffolds下的md文件)
      hexo new "文章标题"
    • 生成一个草稿(不会被渲染和生成到网页上去)
      hexo new draft "草稿标题"
    • 然后生成一个网页
      hexo g
      hexo s
  4. 配置(分为网站_config.yml和主题_config.yml)

二、常用指令速查

  1. 推到远端
    hexo g -d

三、踩坑记录

(1) 图片如何上传？

  • 方式一：picgo，图床方式解决
  • 方式二：相对路径配置

(2) git bash下ctrl+C无效，无法终止node

  • 方式一：git的bug，回退至2.10.0
  • 方式二：windows下kill一个进程
    1. 先查看端口被哪个进程所占用？
      netstat -aon|findstr "4000"
    2. 杀掉这个进程（必须要跳到cmd中才能执行成功）
      taskkill /pid 9352 -t -f
    3. 检查是不是被杀掉了
      tasklist|findstr "9352"

(3) hexo g 时出错 cannot read property ‘utcOffset’ of null

把_config.yml中的时区timezone里的beijing改成Asia/Shanghai

(4) 配置tags和categories

  1. 在hexo根目录下创建tags和categories的页面
    hexo new page "tags"
    hexo new page "categories"

  2. 修改source目录下相应page文件下面的index.md
    type: "tags"和layout: "tags"
    type: "categories"和layout: "categories"

(5) 创建文档时自动打开编辑器的配置

    hexo version(版本3.9)
