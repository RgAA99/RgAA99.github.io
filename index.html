1. line-height相关

  • line-height取15px，1.5em，150%和1.5有什么区别？

    首先line-height是由默认值的，经过测试会随着font-size的值变化，且底线为16px

    设置px是固定值，1.5em和150%和1.5都是相对于当前标签的font-size而言的。

    情况一：如果是inside设置了line-height，那都是相对于inside的font-sizeji计算的，无论这个font-size是不是继承的，三者在此都没有差别。

    情况二：如果inside没有设置line-height，这个line-height是从outside那继承的，此时的百分比和em是一样的，继承的是一个具体的px；而纯数字继承的是一个比例，该比例要乘以当前inside的font-size之后才能得出实际的line-height。

2. &&和||

  • 输出1&&0，0&&1，1||0，0||1

    0,0,1,1

    &&首先计算其左边的表达式，如果它的值为false或可被转换为false(null、NaN、0或undefined),那么将返回左边表达式的值，否则，它将计算右边的表达式， 并返回这个表达式结果作为 &&运算的结果。
    ||首先计算其左边的表达式，如果它的值不为false或不可被转换为false(null、NaN、0或undefined),那么将返回左边表达式的值，否则，它将计算右边的表达式， 并返回这个表达式结果作为||运算的结果。

3. 作用域

  • 什么是作用域？

    作用域是一套规则，用于确定如何查找变量（标识符）的规则，在ES6之前，只有函数作用域和全局作用域

  • 什么是作用域链？

    在查找一个变量的时候，首先是在本函数作用域中查找，没找到就去外层作用域中找，如此反复直到全局作用域。 就像是顺着一条链条从里往外查找变量，这条链条就是作用域链。

  • 什么是词法作用域？

    词法作用域是作用域的一种比较主流的工作模型（还有动态作用域）， 所谓的词法作用域就是在写代码的时候确定了标识符的位置，是静态的作用域 。

  • js的编译过程

    在代码执行之前从上到下的进行编译，当遇到某个用var声明的变量的时候，先检查在当前作用域下是否存在了该变量。如果存在，则忽略这个声明；如果不存在，则在当前作用域中声明该变量。 所以才会有变量提升，在提前访问变量时不出错而是输出undefined。

6.什么是表达式

可以简单理解为一种一定有返回值的语法结构

分类：

  • 单值表达式：没有运算符参与

    • 简单表达式：无法再被分解

      举例：

      this关键字，

      变量，

      普通类型字面量（true，1，”haha“等）

    • 复杂表达式：没有运算符参与，但是由多种简单表达式构成

      举例：

      引用类型的字面量（{a:1,b:2}，[1,2,3])，

      函数定义（不是函数申明，函数定义才有返回值），

      属性调用（如果把”.“看作运算符的话也可以加入复合表达式之中）

      方法调用（func（arg1，arg2））

  • 复合表达式：有运算符参与

    举例：

    typeof ”haha“，

    3+5/2，

    false && true，

    1？true ：false，

    x = 5 （返回值是5）

跨域问题总结

发表于 2019-10-31

一、什么是同源策略？

同源：是指两个URL的协议，域名，端口完全相同。

  • 域名localhost和127.0.0.1虽然都指向本机，还是属于跨域。

同源策略：从一个源上加载的脚本不允许访问另一个源的文档内容（#document），而所谓“跨域”其实是指“跨源”。

什么时候可能会访问到不同的源：

  • 引入：同源策略对于资源如img，script，css和iframe等的访问做了限制，即可以加载引入，但JS不能读写加载的内容。

  • 跳转：页面中的超链接，重定向和表单提交不会受到同源策略的限制。

  • ajax：a页面通过ajax访问一个url，而两者并不同源。

二、同源策略的作用

如果没有同源策略，用户信息可能被不当获取：

  • 在a页面通过iframe标签嵌入b页面，如果没有同源策略，a中的脚本就可以访问b页面中的数据（操作b的dom），同理b中的脚本也能访问a中的数据。假设a在表单中输入账号密码，b可以绑定相应事件获取a的输入。实际情况是

  • 假设用户访问了网站a，a的服务器给浏览器写入了a的相关信息，存在cookie中。用户又访问了网站b，没有同源策略，b中脚本是可以获取到a的cookie的，实际情况是b只能访问同域（路径和父路径）和父域下（路径和父路径）的cookie。（待确认）

三、如何跨域

为什么需要跨域？

比如同一个公司的资源分布在不同的子域中，a.company.com需要访问b.company.com，这个时候就需要跨域。

常见的跨域类型和方法

  1. ajax

    (1) jsonp

    • 原理：本地定义一个函数，被script标签加载的脚本与当前文档看作属于同一个域，所以可以用script标签取得函数的执行语句，通过键值对把变量传给服务器，服务器通过函数参数把变量传回来实现跨域通信。直接返回函数执行，免去了对script异步监听的过程。

    • 缺点：
      1.只能是get不能是post（标签性质决定）
      2.所有的网站都能通过src获取服务器变量，需要协商验证方式

    (2) 后台配置cors(cross-origin resource sharing)

    • 原理：利用自定义的HTTP头部让浏览器和服务器进行沟通，后台进行配置，兼容的浏览器会在前端自动加上HTTP头。

    • 兼容性：IE8以下不支持，IE8和IE9支持但有限制

  2. iframe

    (1) 设置document.domain

    • 适用对象：iframe，cookie

    • 使用限制：只能把document.domain设置成自身或更高一级的父域，且主域必须相同

    • 缺点：
      1.company.com如果被攻击，子域全都可能被攻击
      2.只适用于不同子域之间iframe的交互

    (2) 使用window.name属性进行通信

    • 原理：window对象有个name属性，该属性有个特征：即在一个窗口(window)的生命周期内,窗口载入的所有的页面都是共享一个window.name的，每个页面对window.name都有读写的权限，window.name是持久存在一个窗口载入过的所有页面中的，并不会因新页面的载入而进行重置。
    • 注意：
      1.window.name只能是字符串的形式，最大大概2M左右
      2.将原本子域iframe通信扩展到了可以真正的跨域通信

    （3）使用window.postMessage方式通信（h5）

    • 原理：发送者调用接收者的window对象的postMessage方法向其发消息，接收者调用自己的window对象监听onmessage事件获得收到的消息。

    • 兼容性：IE8+

  3. cookie
    cookie中的同源只关注域名和path，忽略了协议和端口，所以https://localhost:8080/和http://localhost:8081/的Cookie是共享的。

  4. websocket（不受同源策略限制）

    需要服务器的配合，建立连接之后http/https协议会升级为ws/wss

  1. img标签单向通信

    • 原理：和script标签类似，此类标签在发送消息时不受同源策略的影响

    • 缺点：
      1.只能发送get请求
      2.无法访问服务器的响应文本

    • 类似的跨域内嵌标签

      • script
      • link
      • video和audio
      • object，embed和applet
      • @font-face
      • frame和iframe

一、什么是hexo

hexo是基于nodejs的静态网页生成器，可以用来写博客，展示产品等。

(1) 安装过程

  1. 安装node和git
    • git是为了从hexo库中拉取以及将本地部署推到github.io
  2. 按命令执行
    • 安装hexo和相关命令
      npm install -g hexo-cli
    • 在指定 文件夹 下初始化hexo
      hexo init 文件夹
    • 报错后进文件夹重新install
      cd 文件夹
      npm install
    • 生成网站页面(生成在public目录下)
      hexo g
    • 部署至本地server
      hexo s
  3. 写作
    • 创建一篇新文章(默认模板是post,文章的模板参照scaffolds下的md文件)
      hexo new "文章标题"
    • 生成一个草稿(不会被渲染和生成到网页上去)
      hexo new draft "草稿标题"
    • 然后生成一个网页
      hexo g
      hexo s
  4. 配置(分为网站_config.yml和主题_config.yml)

二、常用指令速查

  1. 推到远端
    hexo g -d

三、踩坑记录

(1) 图片如何上传？

  • 方式一：图床方式
    下载picgo，通过上传图片至图床，生成外链，通过![](外链)访问
  • 方式二：在source目录下新建imgs文件
    1. 在图片量较少，便于管理时使用
    2. 通过![](/imgs/图片)访问
  • 方式三：相对路径
    1. 配置根目录_config.yml下post_asset_folder: true
    2. 图片直接放进source下_posts文章同名文件夹中
    3. 通过![](图片)访问，但是主页无法正常显示
    4. 通过 {% asset_img 图片 描述语言 %} 访问最佳

(2) git bash下ctrl+C无效，无法终止node

  • 方式一：git的bug，回退至2.10.0
  • 方式二：windows下kill一个进程
    1. 先查看端口被哪个进程所占用？
      netstat -aon|findstr "4000"
    2. 杀掉这个进程（必须要跳到cmd中才能执行成功）
      taskkill /pid 9352 -t -f
    3. 检查是不是被杀掉了
      tasklist|findstr "9352"

    type: "tags"和layout: "tags"
    type: "categories"和layout: "categories"
