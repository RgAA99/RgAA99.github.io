跨域问题总结

发表于 2019-10-31

一、什么是同源策略？

同源：是指两个URL的协议，域名，端口完全相同。

  • 域名localhost和127.0.0.1虽然都指向本机，还是属于跨域。

同源策略：从一个源上加载的脚本不允许访问另一个源的文档内容（#document），而所谓“跨域”其实是指“跨源”。

什么时候可能会访问到不同的源：

  • 引入：同源策略对于资源如img，script，css和iframe等的访问做了限制，即可以加载引入，但JS不能读写加载的内容。

  • 跳转：页面中的超链接，重定向和表单提交不会受到同源策略的限制。

  • ajax：a页面通过ajax访问一个url，而两者并不同源。

二、同源策略的作用

如果没有同源策略，用户信息可能被不当获取：

  • 在a页面通过iframe标签嵌入b页面，如果没有同源策略，a中的脚本就可以访问b页面中的数据（操作b的dom），同理b中的脚本也能访问a中的数据。假设a在表单中输入账号密码，b可以绑定相应事件获取a的输入。实际情况是

  • 假设用户访问了网站a，a的服务器给浏览器写入了a的相关信息，存在cookie中。用户又访问了网站b，没有同源策略，b中脚本是可以获取到a的cookie的，实际情况是b只能访问同域（路径和父路径）和父域下（路径和父路径）的cookie。（待确认）

三、如何跨域

为什么需要跨域？

比如同一个公司的资源分布在不同的子域中，a.company.com需要访问b.company.com，这个时候就需要跨域。

常见的跨域类型和方法

  1. ajax

    (1) jsonp

    • 原理：本地定义一个函数，被script标签加载的脚本与当前文档看作属于同一个域，所以可以用script标签取得函数的执行语句，通过键值对把变量传给服务器，服务器通过函数参数把变量传回来实现跨域通信。直接返回函数执行，免去了对script异步监听的过程。

    • 缺点：
      1.只能是get不能是post（标签性质决定）
      2.所有的网站都能通过src获取服务器变量，需要协商验证方式

    (2) 后台配置cors(cross-origin resource sharing)

    • 原理：利用自定义的HTTP头部让浏览器和服务器进行沟通，后台进行配置，兼容的浏览器会在前端自动加上HTTP头。

    • 兼容性：IE8以下不支持，IE8和IE9支持但有限制

  2. iframe

    (1) 设置document.domain

    • 适用对象：iframe，cookie

    • 使用限制：只能把document.domain设置成自身或更高一级的父域，且主域必须相同

    • 缺点：
      1.company.com如果被攻击，子域全都可能被攻击
      2.只适用于不同子域之间iframe的交互

    (2) 使用window.name属性进行通信

    • 原理：window对象有个name属性，该属性有个特征：即在一个窗口(window)的生命周期内,窗口载入的所有的页面都是共享一个window.name的，每个页面对window.name都有读写的权限，window.name是持久存在一个窗口载入过的所有页面中的，并不会因新页面的载入而进行重置。
    • 注意：
      1.window.name只能是字符串的形式，最大大概2M左右
      2.将原本子域iframe通信扩展到了可以真正的跨域通信

    （3）使用window.postMessage方式通信（h5）

    • 原理：发送者调用接收者的window对象的postMessage方法向其发消息，接收者调用自己的window对象监听onmessage事件获得收到的消息。

    • 兼容性：IE8+

  3. cookie
    cookie中的同源只关注域名和path，忽略了协议和端口，所以https://localhost:8080/和http://localhost:8081/的Cookie是共享的。

  4. websocket（不受同源策略限制）

    需要服务器的配合，建立连接之后http/https协议会升级为ws/wss

  1. img标签单向通信

    • 原理：和script标签类似，此类标签在发送消息时不受同源策略的影响

    • 缺点：
      1.只能发送get请求
      2.无法访问服务器的响应文本

    • 类似的跨域内嵌标签

      • script
      • link
      • video和audio
      • object，embed和applet
      • @font-face
      • frame和iframe
